# Spec 13 — iPhone PWA Installability

**Status: APPROVED 2026-07-11, not yet implemented.**
**Branch:** `feat/ios-pwa` (off `main`).

## Problem
iOS Safari has no `beforeinstallprompt` event, so the Android-centric PWA flow
doesn't help iPhone users. The app also has 3 missing pieces that prevent iOS
"Add to Home Screen" from producing a proper standalone app: no Apple meta tags,
no apple-touch-icon, and a pre-existing SW scope bug that breaks offline launch
for ALL platforms.

## Changes (5 files + 1 new asset)

### 1. Apple meta tags + theme-color fix — `templates/index.html`
Add to `<head>` after the existing `theme-color` meta:
```html
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
<meta name="apple-mobile-web-app-title" content="Hoarder">
<link rel="apple-touch-icon" href="/static/apple-touch-icon.png">
```
- `apple-mobile-web-app-capable=yes` + `status-bar-style=black-translucent` →
  iOS launches standalone with content extending under the status bar (we
  already have `viewport-fit=cover` + safe-area CSS).
- `apple-mobile-web-app-title=Hoarder` → matches manifest `short_name`.
- **Fix the theme-color mismatch:** head meta is `#101216`, manifest is
  `#0f1115`. Unify both to `#0f1115`.

### 2. Generate apple-touch-icon — `static/apple-touch-icon.png`
Generate a 180×180 PNG from the existing `icon.svg` (same `#0f1115` bg +
`#2dd4bf` glyph). Use Python + Pillow (or `cairosvg` / `rsvg-convert`).
180×180 is the iPhone Pro/Max home-screen icon size. Add it to the SW SHELL
precache list.

### 3. Fix SW scope bug — `web.py` + `static/browse/main.js`
**The bug:** SW registered at `/static/sw.js` with default scope `/static/`.
It cannot intercept navigations to `/`, so the offline navigation fallback
never fires. Affects ALL platforms, not just iOS.

**Fix (route approach — SW file stays physically where it is):**

**`web.py`** — Add a `/sw.js` route:
```python
@app.get("/sw.js")
def service_worker():
    resp = send_from_directory(app.static_folder, "sw.js",
                               mimetype="text/javascript")
    resp.headers["Service-Worker-Allowed"] = "/"
    return resp
```

**`static/browse/main.js`** — Change registration:
```js
navigator.serviceWorker.register("/sw.js", { scope: "/" })
```
(Update the URL from `/static/sw.js` to `/sw.js`, add the scope option.)

### 4. Bump SW cache version — `static/sw.js` + `static/browse/main.js`
- `CACHE` v118 → v119 (SHELL gains `apple-touch-icon.png`)
- `APP_VERSION` v118 → v119 in lockstep

### 5. HTTPS serving via mkcert — `cli.py` + `config.py`
Add `--tls` flag to the `serve` command. When set:
- Look for `cert.pem` + `key.pem` in the project root (generated by mkcert).
- Pass `ssl_context=('cert.pem', 'key.pem')` to `app.run()`.
- Print `https://` instead of `http://` in the startup message.
- Add `CONTENT_HOARDER_TLS` env var to `config.py` defaults (default
  empty = off).

**Usage:**
```bash
# One-time: install mkcert + generate certs
mkcert -install                          # installs local CA in trust store
mkcert -cert-file cert.pem -key-file key.pem \
  0.0.0.0 localhost <tailscale-ip> <tailscale-hostname>

# Serve over HTTPS
python -m content_hoarder serve --host 0.0.0.0 --tls
```

**iPhone setup:**
1. Install mkcert root CA: airdrop `mkcert -CAROOT`'s `rootCA.pem` → iPhone →
   Settings → Profile Downloaded → Install → Settings → General → About →
   Certificate Trust Settings → enable.
2. Navigate to `https://<tailscale-hostname>:8788`.
3. Share → Add to Home Screen → launches standalone with Hoarder icon.

## Out of scope (Icebox)
- **Splash screen images** (`apple-touch-startup-image` for each device size) —
  iOS shows a white/blank screen on launch without these, but the app loads
  fast enough that this is cosmetic. Defer until after initial testing.
- **Install button flow** (`beforeinstallprompt`) for Android — separate
  feature, not needed for iOS.
- **Hosting beyond LAN/Tailscale** — the user serves locally; no public
  hosting needed.
