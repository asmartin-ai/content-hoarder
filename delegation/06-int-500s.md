# Guard the /reddit/sync max_pages parse

## Model: devstral

`POST /reddit/sync` in `src/content_hoarder/web.py` does `int(mp)` on a user-supplied JSON
value — a malformed value (`"abc"`, `[]`) raises an unhandled `ValueError`/`TypeError` → 500.

## Context — current code (src/content_hoarder/web.py)

```python
def _int(value, default: int = 0) -> int:
    try:
        return int(value)
    except (TypeError, ValueError):
        return default
```
```python
    @app.post("/reddit/sync")
    def reddit_sync_route():
        from content_hoarder import reddit_sync
        body = request.get_json(silent=True) or {}
        mp = body.get("max_pages")
        full = bool(body.get("full"))
        max_pages = int(mp) if mp else (50 if full else 3)
        with conn() as c:
            res = reddit_sync.sync_saved_cookie(c, max_pages=max_pages, stop_on_known=not full)
        return jsonify(res)
```

## Requirements

1. Replace the parse with the existing `_int` helper:
   ```python
   max_pages = _int(mp, 0)
   if max_pages <= 0:
       max_pages = 50 if full else 3
   max_pages = min(max_pages, 200)   # hard ceiling — ~200 req @ ~1/s is already extreme
   ```
2. Test in `tests/test_web.py` (style: `web.create_app(tmp_db)` + `test_client()`):
   `POST /reddit/sync` with `{"max_pages": "garbage"}` → 200 (the unconfigured test DB
   yields `auth_error: True`; the point is no 500).

## Constraints

- Do not touch `/reddit/unsave/drain` (separate prompt, delegation/05).
- `sync_saved_cookie` is untouched.

## Acceptance
